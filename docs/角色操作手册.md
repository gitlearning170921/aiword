# AI Word 操作手册（简明版）

> 面向**项目经理（项目管理员）**与日常使用人员。按步骤操作即可；详细验收见《系统测试手册》。  
> 看不到按钮、进不了页面 → 先看 **第七节常见问题**，或联系系统管理员。

---

## 一、项目经理：3 分钟知道用哪些页

| 你要做的事 | 去哪个页面 |
|------------|------------|
| 录入任务、分配给编写人 | **页面1 · 任务上传** |
| 看进度、钉钉催办 | **页面3 · 统计** |
| 对 Word 做 **AI 审核** | **文档审核**（从页面1 进） |
| 按模板 **AI 生成初稿** | **初稿生成**（从页面1 进） |
| 按审核意见自动改文档 | **审核后修改** |
| 观察编写人任务（不能代改） | **页面2 · 我的任务**（只读） |
| 组织考试、看成绩 | **考试训练中心**（老师/统计端） |

**登录**：用户名 + 密码 → 默认进入 **页面1**。  
**前提**：账号已在页面4 绑定 **所属项目组**，否则页面1/3 不可用。

**顶部灰条**：显示当前公司、项目组；列表为空时会写原因（未绑组等）。

## 二、项目经理：派任务与催办

### 2.1 派一个新任务（页面1）

1. 打开 **页面1**，选好 **项目名称**（须已绑定 **所属项目组**）。
2. 填写文件名称、任务类型、截止日期、编写人员等。
3. 点 **保存全部** → 编写人在 **页面2** 能看到该任务。
4. 可选：保存时发钉钉通知（单位已配钉钉时）。

**编写人下拉是空的？**  
→ 项目是否绑了项目组；编写人是否已加入该项目组（找管理员在页面4 维护）。

### 2.2 看进度、催办（页面3）

1. 打开 **页面3 · 统计**（数据仅限 **你所属项目组**，不是全公司总数）。
2. 看完成率、按人/按项目明细。
3. 需要催办：按 **项目**、**人员** 或 **单条任务** 发钉钉。
4. 可在页面3 配置定时催办时间；不同项目组可配不同钉钉群（管理员在页面4 配置）。

### 2.3 从页面0 同步项目

公司管理员在 **页面0** 登记的项目，可在页面1 **同步到本页**，再开始派任务。

## 三、文档审核（项目经理常用）

**作用**：用 AI 检查 Word 是否符合法规、案例与项目要求，生成审核报告。

### 3.1 两种进入方式

| 方式 | 适用场景 | 怎么进 |
|------|----------|--------|
| **带任务审核** | 任务已在页面1，要用任务里的模板/文件 | 任务行点 **单审**，或勾选多条后点 **批量审核** |
| **手动上传审核** | 手头有文件，不在任务列表里 | 页面1 顶部 **文档工具 → 文档审核** |

> 顶部「文档工具」里的审核/初稿是 **自己上传文件**，不会自动带任务列表里的文件。  
> 要对某条任务审，请用任务行的 **单审** 或 **批量审核**。

### 3.2 带任务审核（推荐）

**审 1 个文件**

1. 在页面1 **已保存任务列表** 找到该任务。
2. 点该行 **单审** → 打开「文档审核」页，任务文件已关联。
3. 核对 **所属公司**、**项目**（必填，一般会根据任务自动选好）。
4. 点 **提交审核**，等进度条走完。
5. 下方出现 **审核结果**；可 **下载报告 ZIP**。
6. 需要改报告、标记误报 → 点 **报告编辑**（或在历史记录里进）。

**一次审多个文件**

1. 勾选 **2 条及以上** 任务（同一项目更稳妥）。
2. 点 **批量审核**：
   - 默认：**单文档审核**（每个文件单独审，可多条）；
   - 下拉可选：**多文档一致性**（至少 2 个文件，查互相是否一致）；
   - **跨文档可追溯性**（至少 2 个文件，查追溯矩阵等）。
3. 在新打开的审核页确认设置 → **提交审核**。

### 3.3 手动上传审核

1. 页面1 → **文档工具 → 文档审核**。
2. **来源**：点 **选择文件**（可多次追加；支持 zip 压缩包）。
3. **审核设置**：
   - **审核模式**：一般选「单文档审核」；
   - **所属公司**：选对，决定用哪家公司的知识库；
   - **项目**：**必选**，从下拉选对应 AI 项目；
   - 注册国家/类型等：按项目实际情况选；
   - 「自动匹配过往项目案例」：一般保持勾选。
4. **提交审核** → 等结果 → 下载报告 / 看历史。

### 3.4 审核结果怎么用

- **审核结果区**：列出每条问题（位置、描述、法规依据、修改建议等）。
- **报告编辑**：误报可 **纠正并入库**，以后审核会参考；可把处理状态改为「立即修改 / 延期 / 无需修改」。
- **审核后修改**：处理状态为「立即修改」的项，可再进 **审核后修改** 页，让 AI 按报告改 Word（见 3.5 节简表）。
- **我的审核历史**：可查近次任务、报告编号，重新下载或打开报告编辑。

### 3.5 个人 AI 密钥（可选）

- 在 **初稿生成** 页可保存 **个人 LLM 设置**（API Key）；**审核、初稿、翻译、审核后修改共用**。
- 单位若已配系统默认 Key，可不填；若提示须个人 Key，保存后再提交审核。

### 3.6 审核常见问题

| 现象 | 处理 |
|------|------|
| 没有「单审 / 批量审核」按钮 | 找管理员开「页面1 · 文档审核」权限 |
| 提示未选项目 | 在审核页下拉选 **项目** |
| 报错或一直转圈 | 看顶部 **所属公司** 是否选对；联系管理员查 AI 服务是否可用 |
| 多文档模式报错 | 至少选 **2 个** 任务或 **2 个** 文件 |

## 四、初稿生成（项目经理常用）

**作用**：按知识库里的 **项目案例模板**，结合你上传的参考材料，AI 生成注册文档初稿（Word）。

### 4.1 两种进入方式

| 方式 | 说明 |
|------|------|
| **页面1 文档工具 → 初稿生成** | **手动上传**参考文件，自己选案例和模板（不绑任务列表） |
| **页面2 任务行 → 初稿生成** | 编写人用得多；项目经理在页面2 只读时也可点，会 **自动带入** 项目/产品/国家/文件名 |

项目经理日常更常用：**页面1 顶部 → 初稿生成**。

### 4.2 操作步骤（页面1 进入）

**第一步：准备（首次可选）**

1. 打开 **初稿生成** 页。
2. 若单位要求个人 Key：在 **个人 LLM 设置** 里选提供方、填 API Key → **保存** → 可点 **测试 Key**。
3. 点 **刷新列表**，加载案例、模板、项目下拉。

**第二步：生成设置**

1. **所属公司**：选对（决定用哪家知识库）。
2. **选择模板项目案例**：选与目标注册类型接近的 **案例**（可搜索筛选）。
3. **待生成模板文件范围 / 勾选文件名**：选要生成哪几个 Word（可全选案例下全部模板）。
4. **生成文档语言**、**生成策略**（新项目 / 注册变更等）：按实际情况选。
5. **项目模式 → 选择已有项目**：选 AI 侧对应 **项目**（与审核页类似，必填类信息）。

**第三步：上传文件并提交**

1. **上传输入/参考文件**：产品说明、旧版文档、表格等（Word / Excel / PDF / zip 均可，可多次追加）。
2. 若要在 **已有 Word 上改**（就地修改）：上传 **Base 基础文件**（文件名建议与模板目标名一致）。
3. 可选：在 **自定义提示** 里写「必须改哪一章、不要删哪些段落」等。
4. 点 **提交生成** → 看进度条。
5. 成功后点 **下载 ZIP**，解压得到生成的 docx。

**第四步：历史记录**

- 页底 **初稿生成历史** 可找回近期任务、再次下载（成功且本机曾下载过的会保留缓存）。

### 4.3 从页面2 任务带入时

- 编写人在 **页面2** 某任务点 **初稿生成**，系统会 **预填** 公司、案例、项目、模板文件名等。
- 你作为项目经理 **观察页面2** 时同样可点，但 **不能替别人保存任务状态**；生成流程与上节相同，提交前请 **核对** 预填项。

### 4.4 初稿常见问题

| 现象 | 处理 |
|------|------|
| 案例/模板列表为空 | 先 **刷新列表**；确认 **所属公司** 正确；案例是否已在页面0 训练入库 |
| 提交失败 | 是否选了案例、模板、项目；是否上传了 **输入文件** |
| 生成内容几乎没变 | 是否选了 **就地修改** 并上传了 **Base**；在自定义提示里写清要改的范围 |
| 没有初稿入口 | 找管理员开「页面1 · 初稿生成」 |

## 五、其他文档工具（简要）

| 功能 | 何时用 | 入口 |
|------|--------|------|
| **审核后修改** | 已有审核报告，要把「立即修改」项写进 Word | 页面1 文档工具；或报告编辑页生成待办 |
| **文档翻译** | 整份或批量翻译 docx | 页面1 文档工具 / 页面2 任务行 |
| **去签字 / 去打印** | 跳转到签批系统 | 页面1 任务行（须开启） |

## 六、考试训练中心（项目经理）

- 入口：**页面1 或 页面3 顶部** → 考试训练中心。
- **老师端**：录题、下发考试、阅卷。
- **统计端**：看活动、及格率、薄弱项。
- 操作前在页顶选好 **公司、项目组**；下发时学员列表来自 **当前项目组**。
- 学员从 **页面2** 进 **学生端** 答题。

---

## 七、编写人（简明）

1. 登录 → **页面2 · 我的任务**。
2. 只做 **指派给自己** 的任务：填占位符生成 Word，或上传文件，改 **完成状态**。
3. 任务行可用：**初稿生成、审核、翻译** 等（看管理员是否开启）。
4. 右下角 **问题反馈** 可报障；已结案反馈可 **再次提交**。

---

## 八、公司管理员（简明）

- 只用 **页面0**：登记项目、**知识库训练**（法规/案例上传）。
- 不能派任务、不能看页面2/3。
- 训练前选对 **所属公司**。

---

## 九、系统管理员（简要）

- **页面4** 需访问密码：账号、公司、项目组、功能开关、钉钉、**问题反馈处理**。
- 给项目经理开权限时：**必须绑定所属项目组**；按需开「文档审核 / 初稿生成」等。

---

## 十、一条业务线（谁先后做什么）

1. **公司管理员**（页面0）登记项目、训练知识库。  
2. **项目经理**（页面1）派任务给编写人。  
3. **编写人**（页面2）写文档、标记完成；或用 **初稿/审核** 辅助。  
4. **项目经理**（页面3）看统计、催办；或对稿件做 **审核 → 审核后修改**。  

---

## 十一、常见问题

| 现象 | 怎么办 |
|------|--------|
| 进不了页面1/3 | 是否绑了 **所属项目组** → 找管理员 |
| 页面2 没任务 | 是否指定你为编写人/负责人 |
| 编写人下拉为空 | 项目绑组 + 人员加入该组 |
| 单审/初稿按钮没有 | 管理员开功能开关 + 账号权限 |
| 审核/初稿报错 | 顶部 **所属公司**、审核页 **项目** 是否选对 |
| 案例列表空 | 页面0 是否已训练；公司是否选对 |
| 催办进错钉钉群 | 页面1 项目 **所属项目组** 是否与 Webhook 组一致 |
| 改权限后没变化 | **刷新页面** 或重新登录 |
| 列表一直空 | 看顶部灰条提示 |
