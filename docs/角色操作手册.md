# AI Word 角色操作手册

> 面向日常使用人员：按账号角色说明能做什么、在哪里做、常见前置条件与排错。
>
> 与《系统测试手册》互补：本手册侧重**操作步骤**，测试手册侧重**验收用例**。

## 1. 登录与首页

### 1.1 登录入口

- 地址：`/login`
- 使用**用户名 + 密码**登录（公司管理员、项目管理员、普通用户均走此入口）。
- 登录成功后自动进入**该角色默认首页**，不再使用 URL 里的 `next` 参数跳转。

### 1.2 超级管理员（页面4 访问密码）

- 在**页面4 · 系统管理**输入访问密码（`PAGE13_ACCESS_PASSWORD`）验证。
- 验证后可免账号登录访问全站；退出超级管理员会话后，仍可按账号角色使用系统。
- **仅超级管理员**可进入页面4、维护系统配置与账号 CRUD。

### 1.3 各角色默认首页

| 角色 | 默认首页 | 路径 |
|------|----------|------|
| 超级管理员（访问密码） | 页面4 · 系统管理 | `/admin` |
| 公司管理员 | 页面0 · 公司总览 | `/company` |
| 项目管理员 | 页面1 · 上传 | `/upload` |
| 普通用户 | 页面2 · 我的任务 | `/generate` |

### 1.4 顶部导航可见范围

| 导航 | 超级管理员 | 公司管理员 | 项目管理员 | 普通用户 |
|------|:----------:|:----------:|:----------:|:--------:|
| 页面0 公司总览 | ✓（开关开时） | ✓ | — | — |
| 页面1 上传 | ✓ | — | ✓（已分配项目组） | — |
| 页面2 生成 | ✓ | — | ✓（只读观察） | ✓ |
| 页面3 统计 | ✓ | — | ✓（已分配项目组） | — |
| 页面4 系统管理 | ✓ | — | — | — |

> **项目管理员未分配「所属项目组」时**：无法使用页面1/3，登录或访问相关 API 会提示先在页面4 分配项目组。

## 2. 角色一览

系统有两层身份：

1. **超级管理员**：页面4 访问密码（与账号无关，权限最高）。
2. **账号分级角色**（`admin_role`）：

| 分级角色 | 代码值 | 典型人员 | 数据范围 |
|----------|--------|----------|----------|
| 公司管理员 | `company` | 注册/体系负责人 | 页面0；按**注册国家**维度过滤公司项目 |
| 项目管理员 | `project` | 项目经理、文控 | 页面1/3 + 考试老师/统计；**所属项目组**内项目与任务 |
| 普通用户 | `none` | 编写人、审核人 | 页面2 **指派给自己**的任务；考试学生端 |

另：**所属公司**、**所属项目组**、**12 项功能权限**在账号上单独配置，与分级角色叠加生效（见第 8 节）。

## 3. 超级管理员操作手册

**入口**：页面4 `/admin`（须先通过访问密码）。

### 3.1 字典

| 功能 | 操作 |
|------|------|
| 注册国家字典 | 维护国家名称；页面0/1 下拉引用；已绑定数据不可删 |
| 项目组字典 | 新建项目组；**编辑时关联一个或多个公司**（多对多）；账号「所属项目组」从此选择 |

### 3.2 公司管理

- 新建/编辑公司：名称、slug、**知识库 collection**（对应 aicheckword 向量库）。
- 删除公司前系统会检查是否仍被项目引用。

### 3.3 账号管理

创建或编辑账号时需配置：

| 字段 | 说明 |
|------|------|
| 分级角色 | 无 / 项目管理员 / 公司管理员 |
| 所属公司 | 可多选；影响集成工具、考试中心公司 scope |
| 所属项目组 | **项目管理员必填至少一个**；普通用户建议绑定所在组 |
| 页面0 · 注册国家维度 | 公司管理员用于过滤页面0 项目 |
| 功能权限 | 12 项，分「页面1」「页面2」两组（见 8.2） |

支持：**批量功能权限**（多选账号后统一设置）。

### 3.4 配置管理

- 任务类型、完成状态、审核状态等业务字典（影响页面1/2 下拉选项）。

### 3.5 系统与钉钉

| 分区 | 内容 |
|------|------|
| **页面1 功能开关** | 页面0/1/3 文档工具、任务列表签字/打印/单审、页面1/3 考试入口 |
| **页面2 功能开关** | 页面2 任务行按钮（上传/替换、初稿、审核后修改、翻译）、页面2 考试入口 |
| 基础与安全、催办与钉钉、考试训练中心、aicheckword 集成、aiprintword 签字/打印 等 | 见各分区说明 |

> 页面1 与页面2 开关**相互独立**。仅开页面2 初稿时，页面1 不会出现初稿入口（除非从未保存过页面1 开关且存在旧版回退配置）。

### 3.6 作用域诊断

- 查看当前 session：角色、公司、项目组、知识库 collection、空数据原因等，便于排查「列表为空」。

## 4. 公司管理员操作手册

**首页**：页面0 · 公司总览 `/company`  
**不可访问**：页面1、2、3、4、考试中心（直接访问 URL 会被拒绝或重定向回页面0）。

### 4.1 公司项目总览

- 登记、编辑公司维度项目（名称、注册国家/类别、优先级、所属项目组等）。
- 数据范围：账号配置的 **注册国家 scope** 内的项目。
- 可将页面0 项目**关联到页面1 项目**（同步文控任务体系）。

### 4.2 知识库训练（aicheckword）

入口：页面0 → **知识库训练** 卡片。

| Tab | 用途 |
|-----|------|
| 文件训练 | 上传法规/程序/项目案例等到当前公司的 knowledge collection |
| 审核点 | AI 生成审核点 JSON 并训练入库 |
| 知识库管理 | 刷新统计、清空当前公司库、目录训练 |

操作前在页面顶部 **选择所属公司**；训练数据按公司 collection 隔离。

### 4.3 文档工具（页面0 入口）

若系统开启 **页面1 功能开关** 中对应项，页面0 可进入：

- 初稿生成、文档审核、审核后修改、文档翻译  

（与页面1 共用同一套 page1 开关与账号「页面1」功能权限。）

### 4.4 日常注意

- 不能给编写人**下发页面1 任务**；任务分配由**项目管理员**在页面1 完成。
- 不能查看页面3 统计看板。
- 需在页面4 由超管配置：**所属公司**、**注册国家维度**。

## 5. 项目管理员操作手册

**首页**：页面1 · 上传 `/upload`  
**前提**：账号已分配至少一个 **所属项目组**。

### 5.1 页面1 · 任务与项目管理

#### 5.1.1 项目元数据

- 维护项目优先级、状态（进行中/已结束）、**所属公司**。
- 项目须绑定 **所属项目组**（`assignedTeamId`），编写人员下拉才显示该组成员。
- 已有页面1 任务的项目：所属公司可能被锁定（仅超管可改）。

#### 5.1.2 任务录入

1. 选择或新建**录入块**，选定**项目名称**（进行中项目）。
2. 填写文件名称、任务类型、所属模块、截止日期等。
3. **编写人员**：从该项目绑定项目组的成员中选择（**含本人**）；须先在页面4 把人员加入该项目组。
4. **保存全部**：写入任务库；编写人在页面2 可见对应任务。

> 快速「+」创建账号**仅超级管理员**可见；项目管理员应联系超管在页面4 建号并绑定项目组。

#### 5.1.3 已保存任务列表

- 筛选、编辑、批量修改、分组查看。
- 任务行操作（须全局开关 + 账号权限同时允许）：
  - **单审** → 文档审核
  - **去签字 / 去打印** → aiprintword
- 顶部 **文档工具**：初稿、审核、审核后修改、翻译（页面1 开关 + 页面1 功能权限）。

#### 5.1.4 从页面0 同步项目

- 可将页面0 公司总览中、所属项目组下的项目**同步到页面1**（仅新建尚未关联的项）。

### 5.2 页面2 · 我的任务（只读观察）

- 可查看**所属项目组范围内**全部任务（含他人任务）。
- **只读**：不能改完成状态、不能上传替换、不能提交占位符生成。
- 用于跟催、核对进度；实际编写由普通用户在页面2 操作。

可选：顶部筛选 **项目组 / 人员**（观察模式）。

### 5.3 页面3 · 统计

- 整体完成率、按编写人/项目统计、明细表、催办（钉钉）。
- 数据范围：与页面1 一致，**本所属项目组**内项目。
- **考试训练中心**入口：与页面1 共用 **页面1 考试开关**（`FEATURE_PAGE1_EXAM_CENTER`）。

### 5.4 考试训练中心（老师 / 统计）

| 端 | 入口 | 能力 |
|----|------|------|
| 老师端 | `/exam-center?role=teacher` 或页面1 顶部 | 录题、下发考试、阅卷 |
| 统计端 | `/exam-center?role=analytics` 或页面3 顶部 | 学生活动、及格率等 |

- 顶部切换 **公司、项目组**；下发考试时人员列表为**当前项目组内成员**（不要求每人都有「所属公司」直绑）。
- 须开启 **页面1 功能开关 · 考试训练中心**，且账号 **页面1 · 考试训练中心** 未被禁止。

### 5.5 页面4 相关（项目管理员本身不能进）

- 不能创建/删除账号、不能改系统配置。
- 可联系超管：分配项目组、调整 12 项功能权限、开启/关闭页面1/2 功能开关。

## 6. 普通用户操作手册

**首页**：页面2 · 我的任务 `/generate`

### 6.1 页面2 · 我的任务

- 仅显示 **编写人员或负责人为自己** 的任务（且在所属项目组范围内）。
- 常用操作：
  - 更新**完成状态**
  - 填写**占位符**并生成 docx
  - **上传/替换**模板（页面2 功能开关开启时）
  - 任务行：**初稿 / 审核后修改 / 翻译**（各看页面2 开关 + 账号「页面2」权限）

顶部快捷入口（页面2 开关 + 页面2 权限）：

- 初稿生成、进入考试训练中心（学生端）

### 6.2 文档集成工具

从页面2 任务行或顶部进入（需对应开关与权限）：

| 工具 | 说明 |
|------|------|
| 初稿生成 | 跳转 `/draft-gen`，可带 upload 上下文 |
| 文档审核 | `/audit` |
| 审核后修改 | `/audit-modify` |
| 文档翻译 | `/translate` |

集成请求会带上当前 **所属公司**（scope 条显示），知识库按公司 collection 隔离。

### 6.3 考试训练中心（学生）

- 入口：页面2 顶部「进入考试训练中心」→ `/exam-center?role=student`
- 须 **页面2 功能开关 · 考试训练中心** 开启，且账号 **页面2 · 考试训练中心** 未禁止。
- 完成练习/考试后在页面2 标记任务状态（若项目经理有下发对应任务）。

### 6.4 不可访问

- 页面0、1、3、4  
- 他人未指派给自己的任务  
- 考试老师端、统计端

## 8. 功能开关与账号权限（两层控制）

### 8.1 生效规则

对某一具体功能（如「页面2 · 初稿生成」）：

```
最终是否可用 = 系统配置对应开关为「开」 AND 账号该功能未被「禁止」
```

- **超级管理员（访问密码）**：不受功能开关与账号权限限制。
- 账号选 **跟随系统**：不因该账号单独限制。
- 账号选 **允许**：仍要求系统开关已开。
- 账号选 **禁止**：即使系统开关开着也不可用。

### 8.2 账号功能权限 12 项

**页面1 分组（7 项）**

| 权限项 | 影响范围 |
|--------|----------|
| 初稿生成 | 页面0/1 文档工具入口 |
| 文档审核 | 页面0/1 文档工具、页面1 任务列表「单审」 |
| 审核后修改 | 页面0/1 文档工具 |
| 文档翻译 | 页面0/1 文档工具 |
| 考试训练中心 | 页面1/3 顶部考试入口；老师/统计端 API（与入口角色一致） |
| 去签字 | 页面1 任务列表 |
| 去打印 | 页面1 任务列表 |

**页面2 分组（5 项）**

| 权限项 | 影响范围 |
|--------|----------|
| 上传/替换 | 页面2 任务行 |
| 初稿生成 | 页面2 任务行与顶部 |
| 审核后修改 | 页面2 任务行 |
| 文档翻译 | 页面2 任务行 |
| 考试训练中心 | 页面2 顶部考试入口；学生端 |

> 页面1 与页面2 的「初稿/审核后修改/翻译/考试」**分项独立配置**，可只给编写人开页面2、不给页面1。

### 8.3 系统配置中的功能开关

在页面4 → **系统与钉钉** → **打开系统配置**：

- **页面1 功能开关**：7 个键，填 `1` 开启  
- **页面2 功能开关**：5 个键，填 `1` 开启  

修改后已登录用户**刷新页面**后生效（或重新登录）。

## 9. 典型业务流程

### 9.1 新项目从登记到编写（推荐）

```mermaid
flowchart LR
  A[公司管理员<br/>页面0登记项目] --> B[超管/项目管理员<br/>同步或创建页面1项目]
  B --> C[绑定所属项目组与公司]
  C --> D[项目管理员<br/>页面1录入任务]
  D --> E[指定编写人员]
  E --> F[普通用户<br/>页面2执行并标记完成]
  F --> G[项目管理员<br/>页面3统计催办]
```

### 9.2 知识库 + 审核

1. 公司管理员在页面0 训练法规/案例到**当前公司** knowledge collection。  
2. 编写人在页面2 打开「文档审核」，scope 条确认公司正确。  
3. 提交审核任务；结果在 aicheckword 侧查看。

### 9.3 考试下发

1. 超管开启 **页面1 考试开关**；项目管理员账号允许「页面1 · 考试训练中心」。  
2. 项目管理员进入考试中心老师端，选择 **公司 + 项目组**。  
3. 下发考试时勾选项目组内学员（学员账号须绑定该项目组）。  
4. 学员从页面2 进入学生端答题。

## 10. 常见问题（数据显示 / 权限）

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 登录后进不了页面1/3 | 项目管理员**未分配项目组** | 超管在页面4 账号管理分配「所属项目组」 |
| 编写人员下拉为空 | 项目**未绑项目组**；或组内无成员 | 页面1 项目绑组；页面4 把人员加入该组 |
| 页面2 看不到任务 | 任务未指派给该用户；或用户不在任务项目所在组 | 页面1 检查编写人/负责人与项目组 |
| 按钮/入口不显示 | 系统开关关；或账号功能权限「禁止」 | 页面4 查系统配置 + 账号 12 项权限 |
| 只开了页面2 初稿，页面1 也有 | 从未保存过页面1 开关，旧配置回退 | 在系统配置**显式保存**页面1 初稿为 `0` |
| 考试下发无人可选 | 顶部项目组与学员所在组不一致；或学员未绑组 | 统一项目组；学员账号绑定 team |
| 集成工具报公司/collection 错 | 未选所属公司；账号未绑公司 | scope 条切换公司；超管配置所属公司 |
| 修改权限后仍无变化 | 浏览器未刷新 | 刷新页面或重新登录 |

页面顶部 **作用域条**（ScopeBar）在列表为空时会提示原因（未绑公司、未分配项目组等），可先查看再联系管理员。

## 12. 文档修订说明

- 功能开关已分为 **页面1 / 页面2** 两套；考试入口：页面1/3 用 page1 开关，页面2 用 page2 开关。
