# 生产（Linux Docker）+ 本机全栈测试

## 隔离范围

| 组件 | 本机是否启动 | 数据库是否分环境 | 说明 |
|------|-------------|-----------------|------|
| **aiword** | 是 | **可选**（`FEATURE_ENV_SEPARATION=1` 时 `aiword_test` / `aiword`） | 仅 aiword 本机业务数据可隔离 |
| **aicheckword** | 是 | **否**，始终 `aicheckword` 生产库 | 本机跑新代码联调，数据与生产共用 |
| **aiprintword** | 是 | **否**，始终 `aiprintword_sign` 生产库 | 同上 |

本机测试：**三个服务都在本机启动**（`scripts\start_stack.bat`），aicheckword / aiprintword 用于验证本机代码升级，**库名与生产相同**。

---

## 总开关

```ini
FEATURE_ENV_SEPARATION=0   # 默认：与改前一致，AIWORD_ENV 无效
FEATURE_ENV_SEPARATION=1   # 开启后仅 aiword 按 AIWORD_ENV 分库/分目录
AIWORD_ENV=test            # 本机测试 aiword
```

修改 **aiword** `.env` 后重启 aiword 窗口。`aicheckword` / `aiprintword` 的 `.env` 只配生产库，**不要**设 `AIWORD_ENV` 或 `*_test` 库名。

---

## 本机 `.env` 要点

**aiword `.env`（开启分离时）**

```ini
FEATURE_ENV_SEPARATION=1
AIWORD_ENV=test
MYSQL_HOST=...          # 与生产同一 MySQL
QUIZ_API_BASE_URL=http://127.0.0.1:8000
AICHECKWORD_DRAFT_API_BASE=http://127.0.0.1:8000
AIPRINTWORD_BASE_URL=http://127.0.0.1:5050
```

**aicheckword `.env`** — `MYSQL_DATABASE=aicheckword`（生产库，勿用 `aicheckword_test`）

**aiprintword `.env`** — `MYSQL_DATABASE=aiprintword_sign`，`AIWORD_BASE_URL=http://127.0.0.1:5000`

```powershell

cd F:\wzl\learning\python\aiword\scripts
```
