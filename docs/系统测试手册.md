## 1. 测试环境与前置条件

### 1.1 服务依赖

| 组件 | 说明 |
|------|------|
| **aiword** | 本系统 Web 服务（Flask） |
| **aicheckword** | 知识库训练、审核、初稿、翻译等 AI 能力；aiword 通过 HTTP 调用其 API |

**必须配置**（环境变量或页面4 系统配置，二者以实际部署为准）：

- `AICHECKWORD_DRAFT_API_BASE` 或 `QUIZ_API_BASE_URL`：指向 aicheckword 服务根地址

未配置时，集成页（审核/初稿/翻译）提交任务应出现明确错误提示，而非白屏。

### 1.2 功能开关

在 **页面4 · 系统配置** 或 AppConfig 中可调（键名见 `webapp/app_settings.py`）：

| 开关键 | 作用 | 测试建议 |
|--------|------|----------|
| `FEATURE_COMPANY_REGISTRY` | 启用页面0（公司总览）+ RBAC 按项目组/国家过滤 | **必开**，否则无页面0 与严格权限 |
| `FEATURE_MULTI_TENANT` | 严格多公司隔离；无绑定公司时不静默回退默认公司 | 多公司场景 **必开** |
| `FEATURE_EXAM_CENTER` | 考试训练中心入口 | 测考试中心时开启 |
| `FEATURE_PAGE2_DRAFT_GEN` | 页面2 初稿生成入口 | 测初稿时开启 |
| `FEATURE_PAGE2_AUDIT_MODIFY` | 页面2 审核后修改入口 | 按需 |
| `FEATURE_PAGE2_TRANSLATE` | 页面2 翻译入口 | 按需 |

### 1.3 超级管理员入口

- **页面4** 使用访问密码（环境变量 `PAGE13_ACCESS_PASSWORD`），验证后 session 标记为超级管理员。
- 超级管理员 **无需账号登录** 即可访问全站（含页面4 维护 API）。
- 密码门 API：`POST /api/page13-auth`

### 1.4 推荐测试数据

建议准备 **至少 2 家公司**、**4 个项目组**、**每类角色 1 个账号**：

| 代号 | 用户名（示例） | admin_role | 所属公司 | 项目组 | 国家 scope（公司管理员） | 用途 |
|------|----------------|------------|----------|--------|--------------------------|------|
| SA | （超管密码，无账号） | — | 全部 | 全部 | — | 系统维护、破锁、诊断 |
| CA-A | ca_company_a | company | 公司A | — | 如：中国 | 页面0、知识库训练 |
| PA-A | pa_team_a | project | 公司A | Team-A | — | 页面1/3、考试老师 |
| PA-B | pa_team_b | project | 公司B | Team-B | — | 跨组隔离对比 |
| STU-A | stu_team_a | none | 公司A | Team-A | — | 页面2、考试学生 |
| STU-B | stu_team_b | none | 公司B | Team-B | — | 跨公司/组隔离 |

**公司主数据要求**（页面4 维护）：

- 每家公司有唯一 `knowledge_collection`（如 `regulations`、`company_b_kb`）。
- 创建/更新公司后会同步到 aicheckword（`organizations` ↔ aicheckword `companies`）。

### 1.5 自动化冒烟脚本（可选）

测试前或发版后，开发/CI 可跑：

```bash
# 多租户表结构与 collection 解析（不联真实 aicheckword）
python scripts/multi_tenant_smoke.py

# 考试中心公司/项目组隔离
python scripts/exam_center_smoke.py
```

脚本通过仅表示底层规则未回归；**不能替代** UI 与 aicheckword 联调测试。

## 2. 角色与权限总览

系统存在 **两套身份**：

1. **超级管理员**：页面4 访问密码（`page13_authenticated`）。
2. **账号角色**：`User.admin_role` = `none` | `project` | `company`。

另：`User.is_admin` 为页面2「功能开关豁免」标记，**不改变** admin_role 数据范围。

### 2.1 四类角色对照

| 角色 | 判定 | 默认首页 | 导航可见（功能开关全开时） |
|------|------|----------|----------------------------|
| 超级管理员 | 页面4 密码 | `/admin` | 页面0/1/2/3/4、考试中心、集成工具 |
| 公司管理员 | `admin_role=company` | `/company` | **仅页面0** |
| 项目管理员 | `admin_role=project` 且已分配项目组 | `/upload` | 页面1/3、页面2（只读）、考试中心 |
| 普通用户 | `admin_role=none` | `/generate` | 页面2、考试中心（学生） |

### 2.2 页面 × 角色权限矩阵

| 页面/模块 | 路径 | 超管 | 公司管理员 | 项目管理员 | 普通用户 |
|-----------|------|:----:|:----------:|:----------:|:--------:|
| 页面0 公司总览 | `/company` | 读写 | 读写（国家 scope 内） | **禁止** | **禁止** |
| 页面1 上传 | `/upload` | 读写 | **禁止** | 读写（本组范围） | **禁止** |
| 页面2 生成 | `/generate` | 读写 | **禁止** | **只读** | 读写（本人任务） |
| 页面3 统计 | `/dashboard` | 读写 | **禁止** | 读写（本组范围） | **禁止** |
| 页面4 系统管理 | `/admin` | 读写 | **禁止** | **禁止** | **禁止** |
| 考试中心 | `/exam-center` | 全部 | **禁止** | 老师/统计 | 学生 |
| 文档审核 | `/audit` | 可用 | **禁止**¹ | 可用 | 可用 |
| 初稿生成 | `/draft-gen` | 可用 | **禁止**¹ | 可用 | 可用 |
| 审核后修改 | `/audit-modify` | 可用 | **禁止**¹ | 可用 | 可用 |
| 文档翻译 | `/translate` | 可用 | **禁止**¹ | 可用 | 可用 |

¹ 公司管理员账号若直接访问 URL，会被 `login_required` 拦截并重定向到页面0（API 返回 403 + `needsCompanyAdmin`）。

### 2.3 关键 API 权限分层

| 装饰器 | 允许身份 | 典型 API |
|--------|----------|----------|
| `@super_admin_required` | 仅超管 | 公司 CRUD、项目组 CRUD、账号 POST/PATCH/DELETE、系统配置写 |
| `@page13_access_required` | 超管 + 项目管理员（有项目组） | 页面1 upload、页面3 统计、账号 GET 列表 |
| `@company_registry_api_required` | 超管 + 公司管理员 | 页面0 读 API、训练读、项目组 GET |
| `@company_admin_write_required` | 超管 + 公司管理员 | 页面0 项目写、训练写 |
| `@login_required` | 超管 + 项目管理员 + 普通用户（**不含**公司管理员） | 页面2、集成 API |

### 2.4 登录与跳转用例

| 编号 | 步骤 | 预期 |
|------|------|------|
| ROLE-01 | 公司管理员从登录页正常登录（非页面0 专用入口） | 若未选公司入口，非 company 角色逻辑不适用；company 角色应能登录并进入 `/company` |
| ROLE-02 | 公司管理员登录后访问 `/upload` | 重定向 `/company` 或 403 |
| ROLE-03 | 项目管理员 **未分配项目组** 登录 | 登录失败或访问页面1/3 提示「须先分配项目组」 |
| ROLE-04 | 项目管理员登录后 `next=/dashboard` | 成功进入页面3 |
| ROLE-05 | 普通用户登录后 `next=/admin` | 拒绝，进入 `/generate` |
| ROLE-06 | 超管密码验证后访问任意页面 | 均可访问 |
| ROLE-07 | 项目管理员在页面2 打开他人任务 | 只读，不能保存/提交变更 |

## 3. 公司与知识库（多租户）

### 3.1 概念

- **公司（Organization）**：租户单元；字段 `knowledge_collection` 唯一，对应 aicheckword 侧一个向量库 collection。
- **活跃公司**：Session `active_organization_id`；集成请求通过 Header `X-Aiword-Company-Id` 传给 aicheckword。
- **用户与公司绑定**：`user_organization_memberships`（页面4 账号管理配置）。

### 3.2 数据流（训练/审核/初稿）

```
用户选择公司 → aiword 解析 collection → 调用 aicheckword API（带公司 Header）
→ 数据写入该公司 collection → 切换公司后不可见前公司 job/统计
```

### 3.3 测试用例

| 编号 | 步骤 | 预期 |
|------|------|------|
| ORG-01 | 超管在页面4 新建公司 B，填写 name、slug、knowledge_collection | 列表出现 B；aicheckword 侧有对应 company（若上游可达） |
| ORG-02 | 超管编辑公司 A 的 knowledge_collection | 保存成功；后续 A 的训练/审核走新 collection |
| ORG-03 | 超管删除仍被项目引用的公司 | 应拒绝删除并提示引用 |
| ORG-04 | 公司管理员 CA-A 在页面0 切换公司为 A | scope 条显示 A 名称与 A 的 collection |
| ORG-05 | CA-A 在公司 A 训练入库后，切换为公司 B（若 CA-A 也绑定 B） | B 的「刷新统计」与 A 数据独立 |
| ORG-06 | 普通用户 STU-A 打开 `/audit`，查看公司下拉 | 仅见绑定公司（及项目组关联公司），不见无关公司 |
| ORG-07 | `FEATURE_MULTI_TENANT` 开启，账号无任何公司绑定 | 项目/集成列表为空；scope 条 `emptyReasons` 提示未绑定公司 |
| ORG-08 | 页面0 项目列表选「全部公司（并集）」再选「公司A」 | 并集显示多公司项目；筛选后仅 A |

## 4. 项目组与数据范围

### 4.1 概念

- **项目组（ProjectTeam）**：与 **公司** 多对多（`project_team_organizations`）；人员通过 `user_team_memberships` 绑定。
- **项目管理员**：只能看到 `assigned_team_id` 属于其 `team_ids` 的项目与 upload。
- **普通用户**：页面2 仅 **指派给自己** 的任务，且须在所属项目组范围内。
- **公司管理员**：按 **注册国家**（`user_country_scopes`）过滤页面0 公司项目；**不能**看页面1 upload。

### 4.2 测试用例

| 编号 | 步骤 | 预期 |
|------|------|------|
| TEAM-01 | 超管创建 Team-A、Team-B，分别关联公司 A、B | 页面0 项目组字典可见 |
| TEAM-02 | 超管 PATCH 项目组，修改 `organizationIds` 为多公司 | 页面1 项目可选公司范围随之变化 |
| TEAM-03 | PA-A 登录页面1，查看项目/upload 列表 | 仅 Team-A 数据；Team-B 不可见 |
| TEAM-04 | STU-A 登录页面2 | 仅本人被指派任务；Team-B 任务不可见 |
| TEAM-05 | PA-A 登录页面2，浏览任务列表 | 可看 Team-A 范围内任务，**不可编辑** |
| TEAM-06 | CA-A 设置国家 scope 仅「中国」，页面0 有中国/美国项目 | 仅见中国项目；PATCH 美国项目 → 403 |
| TEAM-07 | 页面1 任务 **已下发** 后，PA-A 尝试改项目组/公司/状态 | 失败（非超管锁） |
| TEAM-08 | 同上任务，超管修改项目组 | 成功 |
| TEAM-09 | 超管在页面0 为 STU-A 修改项目组（PUT `/api/users/<id>/teams`） | 成功；STU-A 刷新后页面2 范围变化 |

## 5. 各页面功能与测试用例

### 5.1 页面0 · 公司总览（`/company`）

## 7. aicheckword 集成端到端

**前提**：aicheckword 已启动；scope 条显示正确的 **所属公司** 与 **knowledge collection**。

集成页均要求：**超管密码** 或 **已登录账号（非公司管理员）**。

### 7.1 文档审核（`/audit`）

| 编号 | 步骤 | 预期 |
|------|------|------|
| INT-A01 | STU-A 选公司 A、aicheckword 项目，上传 docx，single 模式提交 | job 轮询 succeeded；报告摘要可见 |
| INT-A02 | 来源「粘贴文本」提交 | 同步返回审核点列表 |
| INT-A03 | 来源「金山文档链接」提交 | 成功或链接无效时明确错误 |
| INT-A04 | multi 模式仅 1 个文件 | 前端校验失败 |
| INT-A05 | 取消「自动匹配过往案例」后提交 | payload 中 `auto_match_case=false` 生效 |
| INT-A06 | 展开「高级：审核提示词」 | 加载默认 system/user 提示词 |
| INT-A07 | 公司 A 提交后，STU-B（仅公司 B）查历史 | 不见 A 的 job |
| INT-A08 | 从页面1 带 upload_ids 进入审核 | 预填文件名与项目 |

### 7.2 初稿生成（`/draft-gen`）

| 编号 | 步骤 | 预期 |
|------|------|------|
| INT-D01 | 选案例、模板、项目，上传输入文件，提交 | job succeeded；可下载 ZIP |
| INT-D02 | 历史记录点「相同参数」 | 表单回填案例/项目/模板；提示重新选文件 |
| INT-D03 | 未选 aicheckword 项目提交 | 前端或后端校验失败 |

### 7.5 集成失败排查表

| 现象 | 可能原因 |
|------|----------|

## 10. 高风险缺陷检查清单

测试收尾时逐项确认：

- [ ] 公司管理员能打开 `/upload`、`/generate`、`/exam-center`、`/audit`（应全部禁止）
- [ ] 项目管理员 **无项目组** 仍能看到页面1 数据
- [ ] 切换公司后 scope 条 collection 未变，训练/审核写入 **错误库**
- [ ] Team-A 用户能看到 Team-B 的 upload、统计、考试记录、集成 job 历史
- [ ] 页面1 **已下发** 任务被项目管理员改了项目组/公司
- [ ] 多租户开启时，无绑定公司账号仍能看到默认公司数据（应 empty + 提示）
- [ ] 公司管理员 PATCH 越权国家 scope 外项目仍成功
- [ ] aicheckword 宕机时集成页无友好错误、或超长时间无响应无提示
- [ ] 超管与普通用户同时操作同一公司训练「清空知识库」无二次确认
