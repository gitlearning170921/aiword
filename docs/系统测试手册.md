## 1. 测试环境与前置条件

### 1.1 服务依赖

| 组件 | 说明 |
|------|------|
| **AI Word（本系统）** | Web 主站：任务、统计、账号、配置 |
| **aicheckword** | 知识库训练、文档审核、初稿、翻译等 AI 能力（独立服务，本系统通过配置地址调用） |
| **aiprintword**（可选） | 签批/打印；页面1 任务行「去签字 / 去打印」跳转 |

**联调前确认**：

- 页面4 → **系统与钉钉** → **打开系统配置** 中，aicheckword 集成地址已填写且服务可达。
- 未配置或上游不可用时，集成页应出现**明确错误提示**，不应白屏或长时间无响应。

### 1.2 建议开启的功能（按测试范围）

在 **页面4 · 系统配置** 中按需开启（具体项以界面为准）：

| 测试范围 | 建议 |
|----------|------|
| 多公司 / 页面0 | 开启「公司总览 / 多租户」相关开关 |
| 严格公司隔离 | 开启多租户模式（无绑定公司时不应看到其他公司数据） |
| 考试训练中心 | 开启页面1 / 页面2 考试入口开关 |
| 初稿 / 审核 / 翻译 | 开启对应页面1、页面2 功能开关 |

修改系统配置后，测试账号**刷新页面或重新登录**再验。

### 1.3 超级管理员入口

- **页面4 · 系统管理** 需输入**访问密码**（由部署方提供）。
- 验证通过后可管理全站：公司、账号、系统配置、问题反馈处理等。
- 超级管理员可不依赖普通账号登录即可访问各页面（用于破锁与验收）。

### 1.4 推荐测试数据

建议准备 **至少 2 家公司**、**2 个项目组**、**每类角色 1 个账号**：

| 代号 | 角色 | 所属公司 | 所属项目组 | 用途 |
|------|------|----------|------------|------|
| 超管 | 页面4 访问密码 | 全部 | 全部 | 维护、诊断、反馈处理 |
| 公司管理员-A | 公司管理员 | 公司A | — | 页面0、知识库训练 |
| 项目管理员-A | 项目管理员 | 公司A | 项目组甲 | 页面1/3、考试老师/统计 |
| 项目管理员-B | 项目管理员 | 公司B | 项目组乙 | 跨组隔离对比 |
| 编写人-A | 普通用户 | 公司A | 项目组甲 | 页面2、考试学生 |
| 编写人-B | 普通用户 | 公司B | 项目组乙 | 跨公司/组隔离 |

**数据准备要点**：

- 每家公司有独立「知识库」标识（页面4 公司管理中维护）。
- 页面1 项目须绑定**所属项目组**；编写人员来自该项目组成员。
- 账号在页面4 配置：**分级角色**、**所属公司**、**所属项目组**、**12 项功能权限**（见第 2 节）。

### 1.5 自动化冒烟（可选）

发版前后可由开发在服务器执行多租户、考试中心隔离等脚本自检；**不能替代** UI 联调与钉钉实发验收。

## 2. 角色与权限总览

### 2.1 两类身份

1. **超级管理员**：页面4 访问密码，权限最高。
2. **账号分级角色**：公司管理员 / 项目管理员 / 普通用户（在页面4 账号管理中配置）。

另：**12 项功能权限**（页面1 七项 + 页面2 五项）与系统功能开关叠加——某项最终可用 = 系统开关已开 **且** 该账号未被单独禁止。

### 2.2 四类角色对照

| 角色 | 默认首页 | 主要可见导航 |
|------|----------|--------------|
| 超级管理员 | 页面4 | 页面0/1/2/3/4、考试中心、集成工具 |
| 公司管理员 | 页面0 | **仅页面0** |
| 项目管理员 | 页面1 | 页面1/3、页面2（只读）、考试老师/统计 |
| 普通用户 | 页面2 | 页面2、考试学生 |

> 项目管理员**未分配所属项目组**时，无法正常使用页面1/3，系统应提示联系超管分配。

### 2.3 页面 × 角色权限矩阵

| 页面/模块 | 超管 | 公司管理员 | 项目管理员 | 普通用户 |
|-----------|:----:|:----------:|:----------:|:--------:|
| 页面0 公司总览 | ✓ | ✓（国家范围内） | ✗ | ✗ |
| 页面1 任务上传 | ✓ | ✗ | ✓（本组范围） | ✗ |
| 页面2 我的任务 | ✓ | ✗ | 只读 | ✓（本人任务） |
| 页面3 统计 | ✓ | ✗ | ✓（本组范围） | ✗ |
| 页面4 系统管理 | ✓ | ✗ | ✗ | ✗ |
| 考试中心 · 学生 | ✓ | ✗ | ✓ | ✓ |
| 考试中心 · 老师/统计 | ✓ | ✗ | ✓ | ✗ |
| 文档审核/初稿/修改/翻译 | ✓ | ✗¹ | 看开关+权限 | 看开关+权限 |
| 右下角「问题反馈」 | ✓ | ✓ | ✓ | ✓ |

¹ 公司管理员直接打开集成工具 URL 应被拒绝或重定向回页面0。

### 2.4 登录与跳转用例

| 编号 | 步骤 | 预期 |
|------|------|------|
| ROLE-01 | 公司管理员登录 | 进入页面0 |
| ROLE-02 | 公司管理员访问页面1 | 拒绝或重定向页面0 |
| ROLE-03 | 项目管理员未分配项目组登录 | 无法使用页面1/3，有明确提示 |
| ROLE-04 | 项目管理员登录后进入页面3 | 成功，数据限于所属项目组 |
| ROLE-05 | 普通用户尝试进入页面4 | 拒绝 |
| ROLE-06 | 超管密码验证后访问各页 | 均可访问 |
| ROLE-07 | 项目管理员在页面2 打开他人任务 | 只读，不能保存/提交 |
| ROLE-08 | 账号某功能被「禁止」，系统开关为开 | 对应按钮/入口不显示或不可用 |
| ROLE-09 | 仅开页面2 初稿、页面1 初稿关 | 页面2 有入口，页面1 无（除非历史配置回退） |

## 3. 公司与知识库（多租户）

### 3.1 概念（测试视角）

- **公司**：数据与知识库隔离单元；切换公司后，训练/审核/考试任务互不可见。
- **活跃公司**：页面顶部作用域条显示当前公司；集成工具提交时使用该公司知识库。
- **用户与公司**：账号可绑定一个或多个公司（页面4 配置）。

### 3.2 测试用例

| 编号 | 步骤 | 预期 |
|------|------|------|
| ORG-01 | 超管新建公司 B，填写名称与知识库标识 | 列表出现 B；后续训练/审核可独立 |
| ORG-02 | 超管修改公司 A 的知识库标识 | 保存成功；之后 A 的训练走新库 |
| ORG-03 | 超管删除仍被项目引用的公司 | 拒绝并提示引用关系 |
| ORG-04 | 公司管理员在页面0 切换公司 | 作用域条与项目列表同步 |
| ORG-05 | 在公司 A 训练入库后，切换到公司 B | B 的统计与 A 独立 |
| ORG-06 | 编写人打开文档审核，查看公司选择 | 仅见本人绑定/授权公司 |
| ORG-07 | 多租户开启，账号未绑定任何公司 | 列表为空，作用域条提示原因 |
| ORG-08 | 页面0 项目列表「全部公司」与「单公司」筛选 | 并集与筛选结果正确 |

## 4. 项目组与数据范围

### 4.1 概念（测试视角）

- **项目组**：与可多选公司关联；人员通过「所属项目组」绑定。
- **项目管理员**：仅见 `所属项目组` 内的页面1 项目、任务、页面3 统计。
- **普通用户**：页面2 仅**指派给自己**的任务，且须在所属项目组范围内。
- **公司管理员**：按**注册国家**过滤页面0 项目；不能看页面1 任务列表。

### 4.2 测试用例

| 编号 | 步骤 | 预期 |
|------|------|------|
| TEAM-01 | 超管创建两个项目组并关联不同公司 | 页面0/4 可见 |
| TEAM-02 | 超管修改项目组关联公司 | 页面1 可选公司范围随之变化 |
| TEAM-03 | 项目管理员甲登录页面1 | 仅见项目组甲数据 |
| TEAM-04 | 编写人甲登录页面2 | 仅本人任务；项目组乙任务不可见 |
| TEAM-05 | 项目管理员甲在页面2 浏览 | 可看本组任务，不可编辑 |
| TEAM-06 | 公司管理员国家范围仅「中国」 | 仅见中国国家项目；编辑他国项目失败 |
| TEAM-07 | 页面1 任务已下发后，项目管理员改项目组/公司 | 应失败（非超管锁） |
| TEAM-08 | 超管修改已下发任务的项目组 | 成功 |
| TEAM-09 | 超管调整编写人所属项目组 | 编写人刷新后页面2 范围变化 |

## 5. 各页面功能与测试用例

### 5.1 页面0 · 公司总览

**功能**：公司项目登记、关联页面1 项目、知识库训练、文档工具入口（受开关控制）。

| 编号 | 步骤 | 预期 |
|------|------|------|
| P0-01 | 公司管理员新建项目（在授权国家内） | 成功，列表刷新 |
| P0-02 | 编辑优先级、注册国家、所属项目组 | 成功 |
| P0-03 | 关联页面1 已有项目 | 关联成功 |
| P0-04 | 项目管理员访问页面0 | 拒绝 |
| P0-05 | 超管批量维护总览项目 | 成功 |

### 5.2 页面1 · 任务上传

**功能**：模板上传、任务录入、指派编写人/负责人、项目元数据、文档工具与签批入口。

| 编号 | 步骤 | 预期 |
|------|------|------|
| P1-01 | 项目管理员上传 docx 并保存任务 | 成功 |
| P1-02 | 指派给编写人甲 | 编写人甲页面2 可见 |
| P1-03 | 项目管理员尝试新建系统账号 | 无权限或按钮不可见 |
| P1-04 | 筛选所属公司 | 仅见授权范围内项目 |
| P1-05 | 保存任务并勾选发送通知 | 负责人收到钉钉分配通知（已配置 Webhook 时） |
| P1-06 | 项目未绑项目组时打开编写人下拉 | 为空或提示先绑组 |

### 5.3 页面2 · 我的任务

**功能**：占位符填写、生成 docx、完成状态、上传/替换、跳转集成工具。

| 编号 | 步骤 | 预期 |
|------|------|------|
| P2-01 | 编写人生成已指派任务 | 成功下载 |
| P2-02 | 访问未指派任务 | 不可见 |
| P2-03 | 项目管理员进入 | 只读 |
| P2-04 | 从任务行进入「文档审核」 | 携带当前任务上下文 |
| P2-05 | 标记完成状态 | 页面3 统计同步更新 |

### 5.4 页面3 · 统计看板

**功能**：按**所属项目组**统计完成率；手动催办；定时催办配置；模块级联催办状态；按项目组配置钉钉。

| 编号 | 步骤 | 预期 |
|------|------|------|
| P3-01 | 项目管理员甲查看统计 | 仅项目组甲数据，非全站总数 |
| P3-02 | 公司管理员访问 | 拒绝 |
| P3-03 | 按项目 / 按人员 / 单条任务催办 | 消息发到对应项目组机器人或默认群 |
| P3-04 | 点击「测试自动催办」各按钮 | 按真实数据发送，文案含页面2 链接 |
| P3-05 | 修改定时任务时间并保存 | 无需重启，新时间生效 |
| P3-06 | 项目组乙配置独立 Webhook，甲未配置 | 乙的任务进乙群，甲的任务进默认群 |
| P3-07 | 页面1 项目归属项目组与 Webhook 组不一致 | 催办走项目所属组路由，而非误发默认群 |

### 5.5 页面4 · 系统管理

**功能**：字典、公司、账号（含 12 项功能权限）、系统与钉钉、**项目组独立 Webhook**、作用域诊断、**问题反馈处理**。

| 编号 | 步骤 | 预期 |
|------|------|------|
| P4-01 | 非超管访问 | 密码门或拒绝 |
| P4-02 | 创建账号并配置角色/公司/项目组/权限 | 登录行为符合第 2 节 |
| P4-03 | 打开「作用域诊断」 | 显示当前会话、公司、组、知识库等可读信息 |
| P4-04 | 「项目组钉钉」为某组填写与全局不同的 Webhook | 保存成功；页面3 催办走路由 |
| P4-05 | 填写与全局相同的 Webhook | 提示不会单独发群，仍走默认机器人 |
| P4-06 | 「问题反馈」Tab 处理用户反馈 | 可改状态、填处理方案、删除记录 |

## 8. 考试训练中心

## 9. 钉钉催办与按项目组通知

### 9.1 配置项（测试准备）

| 配置位置 | 作用 |
|----------|------|
| 页面4 → 系统配置 → 催办与定时通知 | **全局默认** Webhook（未单独配置的项目组使用） |
| 页面4 → 系统与钉钉 → 项目组钉钉 | 某项目组**独立**机器人（URL 须与全局不同才单独路由） |
| 页面4 → 账号管理 → 手机号 | 催办 @ 编写人/负责人 |
| 页面3 → 自动通知时间 | 周四提醒、逾期前一日、项目统计 cron |
| 系统配置 → 站点根地址 | 催办消息中「页面2」链接可点击打开 |

### 9.2 手动催办（页面3）

| 编号 | 步骤 | 预期 |
|------|------|------|
| DING-01 | 按项目催办 | 消息含项目名（加粗）、未完成列表、页面2 链接 |
| DING-02 | 按编写人催办 | 仅该人员相关任务；多项目组时分条发送 |
| DING-03 | 单条任务催办 | 发到任务所属项目组 Webhook |
| DING-04 | 项目组甲配独立 Webhook，任务属甲 | 消息进甲群，不进默认群 |
| DING-05 | 项目组 Webhook 与全局相同 | 系统提示不单独发群，走默认机器人 |
| DING-06 | 未配置任何 Webhook | 明确提示无法发送 |

### 9.3 自动催办（定时 / 测试按钮）

| 编号 | 步骤 | 预期 |
|------|------|------|
| DING-07 | 周四提醒（测试按钮） | 按项目组分条发送；文案含「项目组：xxx」；统计数为该组任务 |
| DING-08 | 逾期前一日 | 明日截止且未完成任务；按负责人合并 |
| DING-09 | 每两天项目统计 | 按项目组、按项目、按人统计未完成数 |
| DING-10 | 模块级联（产品完成→催开发等） | 按项目所属组路由 Webhook |
| DING-11 | 页面1 保存任务分配通知 | @ 负责人（姓名与钉钉昵称一致时生效） |

## 10. 问题反馈

**入口**：页面0～3 及子页面（含考试中心各端）右下角**浮动按钮**（可拖动）；登录页、页面4 **不显示**。

### 10.1 用户提交

| 编号 | 步骤 | 预期 |
|------|------|------|
| FB-01 | 登录后打开反馈弹窗 | 功能模块默认当前页面；可填优先级、描述、截图 |
| FB-02 | 提交必填项完整 | 成功提示；「我的反馈」可见新记录 |
| FB-03 | 上传超大或非法格式截图 | 明确错误；文字反馈仍可提交（视规则） |
| FB-04 | 未登录点击浮动按钮 | 不应出现或提示先登录 |
| FB-05 | 考试中心学生/老师/统计页 | 模块选项对应当前端（非混为一项） |

### 10.2 我的反馈

| 编号 | 步骤 | 预期 |
|------|------|------|
| FB-06 | 查看历史列表 | 显示状态、模块、描述、处理方案（若有） |
| FB-07 | 状态为已解决/已关闭/不予解决 | 显示「问题仍存在，再次提交」 |
| FB-08 | 点击再次提交 | 跳转提交 Tab，预填模块与描述（带跟进前缀），可修改后提交为新记录 |
| FB-09 | 有截图的反馈 | 可查看截图链接 |

### 10.3 超管处理（页面4 · 问题反馈）

| 编号 | 步骤 | 预期 |
|------|------|------|
| FB-10 | 列表加载 | 提交人显示一行（显示名与用户名相同时不重复） |
| FB-11 | 状态改为：待处理 / 处理中 / 已解决 / 已关闭 / **不予解决** | 保存成功；用户侧状态同步 |
| FB-12 | 填写处理方案 | 用户在「我的反馈」可见 |
| FB-13 | 按状态筛选 | 列表过滤正确 |
| FB-14 | 删除记录 | 确认后删除；用户侧不再可见 |

## 12. 高风险缺陷检查清单

测试收尾时逐项确认：

- [ ] 公司管理员能打开页面1/2/3/4、考试中心、集成工具（应全部禁止）
- [ ] 项目管理员**无项目组**仍能看到页面1 数据
- [ ] 切换公司后知识库未变，训练/审核写入错误公司库
- [ ] 项目组甲用户能看到项目组乙的任务、统计、考试记录、集成历史
- [ ] 页面1 **已下发**任务被项目管理员改了项目组/公司
- [ ] 多租户开启时，无绑定公司账号仍能看到其他公司数据
- [ ] 公司管理员能编辑授权国家外的项目
- [ ] aicheckword 不可用时集成页无友好错误
- [ ] 清空知识库无二次确认
- [ ] 项目组独立 Webhook 已配，任务仍全部进默认群（查项目所属项目组）
- [ ] 页面3 统计「全部/已完成」显示全站数而非本组数
- [ ] 反馈提交人重复显示；不予解决状态不可选；已结案反馈无法再次提交
